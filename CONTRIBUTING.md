# Contributing

Thank you for your interest in contributing to Bedrock Win Counter.

## Contribution model

- The repository is public and contributions are welcome.
- All changes are reviewed by the maintainers before they are merged.
- Opening a pull request does not guarantee inclusion.

## How to contribute

1. Fork the repository.
2. Create a feature branch for your change.
3. Keep your change focused and easy to review.
4. Update documentation if your change affects behavior, commands, permissions, or configuration.
5. Open a pull request with a clear explanation of:
   - what changed
   - why it changed
   - anything that should be tested manually

## Guidelines

- Keep changes compatible with the current plugin purpose and Bedrock TNT Box workflow.
- Do not include unrelated refactors in the same pull request.
- Do not remove existing attribution, license text, or copyright notices.
- If you add new configuration options, make sure defaults are documented.

## License for contributions

By submitting a contribution to this repository, you confirm that:

- you have the legal right to submit the contribution
- the contribution is your own work or that you are permitted to submit it
- you agree that your contribution may be distributed under the repository's license

## Maintainer review

Maintainers may:

- request changes before merge
- reject contributions that do not fit the project direction
- edit or adapt a contribution before release
